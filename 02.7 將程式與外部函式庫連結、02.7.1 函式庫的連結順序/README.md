# 2.7 將程式與外部函式庫連結、2.7.1 函式庫的連結順序
## 常見問題與解答
### 為何沒有加上 -lm 參數編譯沒有發生錯誤？
比較新的 GCC 版本預設會用自己內建的 sqrt 函式取代掉 libm 的 sqrt 函式呼叫來對程式作最佳化(optimization)，故實際上沒有用到 libm 函式庫所以編譯沒發生錯誤。
在編譯命令中加上 -fno-builtin 參數可以強迫 GCC 關閉此功能以避免此問題
